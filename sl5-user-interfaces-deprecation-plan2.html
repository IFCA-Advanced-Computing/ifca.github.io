POSTPONED: SL5 User Interfaces deprecation plan (2)

Due to a mistake with the dates, the deprecation plan has been POSTPONED until April 7th 2014. Please check the new plan below.

After the postponed SLC5 deprecation plan we have restablish the procedure to deprecate the Scientific Linux 5 user interfaces. The operating system on these machines has become obsolete and are going to be removed according to the following deprecation plan.

Deprecation plan

  1. April 7th 2014: gridui.ifca.es will stop pointing to the Scientific Linux 5 machines, and will redirect users to the Scientific Linux 6 machines.
  2. April 14th 2014: griduisl5.ifca.es will become unavailable, therefore there will be no access to the Scientific Linux 5 machines anymore. Please, update your working environment before that date.
  3. April 30th 2014: Deletion and removal of machines.

SSH warnings about host identification changed

Please take into account that on April 31th the fingerprint of gridui.ifca.es will change. It is possible that you will get messages like this:

user@mymachine:~ $ ssh user@gridui.ifca.es

IT IS POSSIBLE THAT SOMEONE IS DOING SOMETHING NASTY!

It is also possible that a host key has just been changed.

Please contact your system administrator.
Add correct host key in /home/user/.ssh/known_hosts to get rid of this message.
Offending RSA key in /home/user/.ssh/known_hosts:58
  remove with: ssh-keygen -f "/home/user/.ssh/known_hosts" -R gridui.ifca.es

Check that the new fingerprint is exactly 29:80:9b:28:e7:8a:00:fe:6c:60:ef:e6:a6:71:33:bd. If it is, then you can safely remove the offending match with the following command:

user@mymachine:~ $ ssh-keygen -f "/home/user/.ssh/known_hosts" -R gridui.ifca.es

Access to legacy SL5 machines

Access to Scientific Linux 5 machines is still provided trough the batch system. In order to request a Scientific Linux 5 machine you must specify the complex scientificlinux5 like this:

user@cloudprv-10-0:~ $ qsub -l scientificlinux5=true (...)

If you want an interactive session, request one with qlogin:

user@cloudprv-10-0:~ $ qlogin -l scientificlinux5=true (...)
